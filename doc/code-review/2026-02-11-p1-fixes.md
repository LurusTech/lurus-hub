## Issue #1: 版本比较逻辑错误 ✅

### 问题描述
`internal/app/release_service.go:145-156`

**原始代码**:
```go
func compareVersions(v1, v2 string) int {
    if v1 == v2 {
        return 0
    }
    if v1 < v2 {  // ❌ 字符串比较
        return -1
    }
    return 1
}
```

**问题**:
- 使用字符串比较导致错误结果
- `"1.10.0" < "1.9.0"` 返回 `true`（错误！）
- `"10.0.0" < "9.0.0"` 返回 `true`（错误！）

### 修复方案

**修复后的代码**:
```go
import (
    "golang.org/x/mod/semver"
    "strings"
)

func compareVersions(v1, v2 string) int {
    // Ensure versions start with 'v' for semver.Compare
    if !strings.HasPrefix(v1, "v") {
        v1 = "v" + v1
    }
    if !strings.HasPrefix(v2, "v") {
        v2 = "v" + v2
    }

    // Validate versions
    if !semver.IsValid(v1) || !semver.IsValid(v2) {
        // Fallback to string comparison for non-semver formats
        if v1 == v2 {
            return 0
        }
        if v1 < v2 {
            return -1
        }
        return 1
    }

    return semver.Compare(v1, v2)
}
```

**改进**:
- ✅ 使用 `golang.org/x/mod/semver` 进行语义化版本比较
- ✅ 自动添加 'v' 前缀（兼容 "1.0.0" 和 "v1.0.0"）
- ✅ 验证版本格式，非 semver 格式回退到字符串比较
- ✅ 支持 prerelease 版本（alpha, beta, rc）

## Issue #2: 类型断言 panic 风险 ✅

### 问题描述
`internal/adapter/handler/alipay.go:223`

**原始代码**:
```go
session := sessions.Default(c)
id := session.Get("id")
user.Id = id.(int)  // ❌ 如果 id 为 nil 会 panic
```

**问题**:
- 未检查 session 是否存在
- 类型断言失败会导致 panic（runtime crash）
- 未登录用户可以触发后续 API 调用（安全风险）

### 修复方案

**修复 1: 添加安全类型断言**
```go
session := sessions.Default(c)
id := session.Get("id")
userId, ok := id.(int)  // ✅ 安全类型断言
if !ok || userId == 0 {
    c.JSON(http.StatusUnauthorized, gin.H{
        "success": false,
        "message": "未登录或会话已过期",
    })
    return
}
user.Id = userId
```

**修复 2: 优化执行顺序（安全最佳实践）**

将 session 检查移到函数开头，避免未登录用户触发 API 调用：

```go
func AlipayBind(c *gin.Context) {
    // Check OAuth enabled
    if !common.AlipayOAuthEnabled {
        // ... return
    }

    // ✅ Check session FIRST before making any API calls
    session := sessions.Default(c)
    id := session.Get("id")
    userId, ok := id.(int)
    if !ok || userId == 0 {
        c.JSON(http.StatusUnauthorized, gin.H{
            "success": false,
            "message": "未登录或会话已过期",
        })
        return
    }

    // Now safe to proceed with API calls
    authCode := c.Query("code")
    alipayUserId, _, err := getAlipayUserInfoByCode(authCode)
    // ...
}
```

**改进**:
- ✅ 安全类型断言（防止 panic）
- ✅ 检查 userId == 0（防止空会话）
- ✅ 返回 401 Unauthorized（正确的 HTTP 状态码）
- ✅ 优化执行顺序（先验证身份，再调用外部 API）
- ✅ 清晰的中文错误提示

## Issue #3: Secrets 泄露 ✅

### 问题描述
`deploy/k8s/secrets.yaml`

**原始内容**:
```yaml

  SQL_DSN: "postgres://lurus:LurusOps2026@100.94.177.10:30543/lurusapi?sslmode=disable"
```

**问题**:
- ❌ 文件包含真实的数据库密码
- ❌ 已提交到 Git 历史（泄露）

- 数据库凭证泄露（Critical）
- Session secret 泄露（High）

**修复 1: 替换为 Placeholder**

```yaml
# 2. Edit secrets.prod.yaml with real values (see 重要信息.md)
# 3. Apply to cluster: kubectl apply -f secrets.prod.yaml
# 4. Add secrets.prod.yaml to .gitignore
#
# For CI/CD: Use Sealed Secrets or External Secrets Operator

stringData:
  # Database connection
  # Format: postgres://USER:PASSWORD@HOST:PORT/DATABASE?sslmode=disable
  SQL_DSN: "postgres://placeholder:placeholder@placeholder:5432/placeholder?sslmode=disable"

  # Session secret (generate with: openssl rand -base64 32)
  SESSION_SECRET: "PLACEHOLDER-REPLACE-WITH-RANDOM-32-CHAR-STRING"

  # Alipay OAuth (RSA2 keys from Alipay Developer Console)
```
